# 官网移动端阅读体验适配

## 目标
优化 VTE 官网在移动端（手机/平板）的阅读和交互体验，解决当前存在的导航缺失、内容溢出、性能浪费等问题。

## 问题清单

| # | 问题 | 影响范围 | 优先级 |
|---|------|---------|--------|
| 1 | **导航菜单缺失**：768px 以下 nav-links 直接隐藏，无替代导航 | 全站 | P0 |
| 2 | **文档侧边栏消失**：Docs 页 900px 以下 sidebar 直接 display:none | Docs | P0 |
| 3 | **页面标题过大**：各页面 page-title 56px，移动端溢出 | 全站 | P1 |
| 4 | **参数表格溢出**：PackageDetail 的 params-table 固定列宽 120px+140px | PackageDetail | P1 |
| 5 | **Hero 粒子动画**：移动端未禁用 canvas 粒子，浪费性能 | Home | P1 |
| 6 | **Configurator 侧边栏**：移动端只是纵向堆叠，无收起/展开机制 | Configurator | P1 |
| 7 | **触摸目标太小**：按钮、链接尺寸未适配触摸操作（需 ≥44px） | 全站 | P2 |
| 8 | **MouseGlow 未禁用**：桌面端鼠标跟随效果在移动端无效但仍在运行 | Home | P2 |

## 实施方案

### 1. Navbar - 移动端汉堡菜单 (`Navbar.vue`)

**改动**：
- 添加汉堡按钮（仅 ≤768px 显示）
- 添加抽屉式移动端菜单（从右侧滑出）
- 点击链接或遮罩层自动关闭
- 菜单内包含所有导航链接 + 主题切换 + GitHub

**关键代码**：
```vue
<button class="hamburger" @click="menuOpen = !menuOpen" v-show="!menuOpen">
  <!-- hamburger icon -->
</button>
<div class="mobile-menu" :class="{ open: menuOpen }">
  <div class="mobile-overlay" @click="menuOpen = false"></div>
  <div class="mobile-drawer">
    <router-link v-for="link in links" @click="menuOpen = false" />
  </div>
</div>
```

### 2. Docs 页 - 移动端浮动目录 (`Docs.vue`)

**改动**：
- 添加浮动 TOC 按钮（右下角圆形按钮，≤900px 显示）
- 点击展开侧边栏为全屏弹窗
- 选择章节后自动关闭
- 保留 IntersectionObserver 自动高亮当前章节

### 3. 全局响应式字体 (`App.vue` 或全局 CSS)

**改动**：在 `App.vue` 的 `:root` 中添加响应式字体变量，各页面 page-title 使用 `clamp()`：
```css
@media (max-width: 768px) {
  .page-title { font-size: clamp(28px, 8vw, 40px); }
  .page-desc { font-size: 16px; }
  .section-title { font-size: clamp(24px, 6vw, 32px); }
}
```

### 4. 参数表格移动端适配 (`PackageDetail.vue`)

**改动**：≤768px 时将 grid 布局改为纵向堆叠：
```css
@media (max-width: 768px) {
  .params-table .param-row {
    grid-template-columns: 1fr;
    gap: 4px;
  }
}
```

### 5. Hero 粒子动画优化 (`Hero.vue`)

**改动**：
- 在 `initCanvas()` 中检测 `window.innerWidth`，≤768px 时跳过粒子初始化
- 或减少粒子数量至 15

### 6. Configurator 移动端优化 (`Configurator.vue`)

**改动**：
- 添加侧边栏切换按钮（≤768px 显示）
- 侧边栏默认隐藏，点击按钮弹出为 overlay
- 选择 token 后自动关闭侧边栏

### 7. 触摸目标尺寸 (全局)

**改动**：在 ≤768px 的 media query 中，确保按钮最小高度 44px：
- `.btn` padding 增大
- `.nav-link` padding 增大
- `.action-btn` padding 增大

### 8. MouseGlow 移动端禁用 (`MouseGlow.vue`)

**改动**：在 `onMounted` 中检测 `matchMedia('(hover: hover)')`，非桌面端不初始化。

## 涉及文件

| 文件 | 改动类型 |
|------|---------|
| `website/src/components/Navbar.vue` | 添加汉堡菜单 + 移动端抽屉 |
| `website/src/views/Docs.vue` | 添加浮动 TOC 按钮 + 弹窗侧边栏 |
| `website/src/views/Home.vue` | 无需改动（Hero 自身处理） |
| `website/src/components/Hero.vue` | 粒子动画移动端优化 |
| `website/src/views/PackageDetail.vue` | 参数表格响应式 |
| `website/src/views/Configurator.vue` | 侧边栏切换 + overlay |
| `website/src/components/MouseGlow.vue` | 移动端禁用 |
| `website/src/App.vue` | 全局响应式字体变量 |

## 验证方式

1. `pnpm dev:website` 启动开发服务器
2. Chrome DevTools 切换到移动端视图（iPhone 14 / iPad）
3. 逐页检查：
   - 导航菜单可打开/关闭/跳转
   - Docs 页可打开 TOC 弹窗并跳转
   - 所有页面标题不溢出
   - PackageDetail 参数表格可正常阅读
   - Hero 页无卡顿（粒子已禁用）
   - Configurator 侧边栏可切换
   - 所有按钮/链接触摸区域 ≥44px
