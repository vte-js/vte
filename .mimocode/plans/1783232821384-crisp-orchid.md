# VTE Token Configurator — 在线配置生成器

## 概述

在 VTE 网站新增一个 **Token Configurator** 页面，用户通过可视化 UI 创建/编辑 design-tokens.ts，支持实时预览、模板系统、多格式导入导出。

与 Playground 的区别：Playground 是"已有 token → 预览调试"，Configurator 是"从零配置 → 导出文件"。

---

## 实现阶段

```
Phase 1: 数据层 → Phase 2: 核心 UI → Phase 3: 预览 → Phase 4: 导入导出 → Phase 5: 收尾
```

---

## Phase 1: 数据层 + 模板

**新建文件：**

| 文件 | 说明 |
|------|------|
| `website/src/composables/useTokenStore.ts` | 响应式 token 树 + CRUD + 引用解析 |
| `website/src/composables/useTokenTemplates.ts` | 模板加载器 |
| `website/src/data/templates.ts` | 4 个内置模板定义 |

**数据模型：**
```typescript
interface TokenNode {
  name: string;
  value: string;       // 叶子值或引用 {path}
  children?: TokenNode[];
  isReference: boolean;
}

const store = reactive({
  tree: { primitive: {}, semantic: {}, component: {} },
  activePath: null as string | null,
  activeLayer: "primitive",
  searchQuery: "",
});
```

**关键逻辑：**
- `flattenTree()` — 递归展平为 `Map<string, TokenValue>`
- `resolveRefs()` — BFS 解析引用（复用 @vte-js/core 的算法）
- `addToken(path, value)` / `removeToken(path)` / `updateToken(path, value)`
- localStorage 持久化（debounce 保存）

**模板：**
- `default` — VTE 默认模板（blue/gray/green/red/yellow）
- `tailwind` — Tailwind 完整色板
- `material` — Material Design 3 色彩系统
- `minimal` — 最小可用模板（3 色 + 2 间距 + 1 字号）

**布局：**
```
┌─────────────────────────────────────────────────────┐
│  [toolbar: 模板 | 导入 | 导出 | 搜索]               │
├──────────┬───────────────────┬──────────────────────┤
│  SIDEBAR │   EDITOR PANEL    │   PREVIEW PANEL      │
```

**编辑器功能：**
- Token 名称输入（支持数字键如 `50`, `100`）
- 值输入（颜色用 color picker，尺寸用 slider + 单位选择）
- 引用切换 — 搜索下拉选择已有 token 路径
- 引用验证 — 绿色✓表示已解析，红色⚠表示断链
- 新增/删除按钮

## Phase 4: 导入导出

**新建文件：**

| 文件 | 说明 |
|------|------|
| `website/src/composables/useTokenImport.ts` | 解析器：JSON / CSS Variables / design-tokens.ts |
| `website/src/composables/useTokenExport.ts` | 生成器：TS / JSON / CSS / SCSS |
| `website/src/components/configurator/TokenImportModal.vue` | 导入弹窗 |
| `website/src/components/configurator/TokenExportModal.vue` | 导出弹窗 |
| `website/src/components/configurator/CodeOutput.vue` | 代码高亮输出（复用 Prism.js） |

**导入：**
- 自动检测格式（JSON 以 `{` 开头，TS 有 `defineTokens`，CSS 有 `--`）
- CSS Variables 解析为 semantic 层
- 导入后替换整个 store

**导出：**
- TypeScript: `export default defineTokens({...})`
- JSON: 标准 JSON 格式
- CSS Variables: `:root { --vte-xxx: value; }`
- SCSS: `$vte-xxx: value;`
- 支持复制到剪贴板 + 下载文件

## 验证方式

1. 启动 website dev server，访问 `/configurator`
2. 选择模板 → 验证 token 树正确加载
3. 添加/编辑/删除 token → 验证实时预览更新
4. 创建引用 → 验证引用解析和验证指示器
5. 导入 JSON/CSS → 验证正确解析
6. 导出为各格式 → 验证代码正确性
7. 暗黑模式切换 → 验证样式适配
8. 刷新页面 → 验证 localStorage 持久化
