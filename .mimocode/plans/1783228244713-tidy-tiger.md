# NPM 发布工作流优化计划

## 目标
将 VTE 的 7 个 npm 包（+ 1 个 VS Code 扩展）配置为可发布状态，并建立自动化发布流程。

## 当前问题

| 问题 | 影响 |
|------|------|
| 无 `files` 字段 | 发布时会包含源码、测试等无关文件 |
| 无 `publishConfig` | scoped 包 @vte-js/* 默认 private，发布会被拒绝 |
| 无 `exports` 字段 | ESM/CJS 双格式支持不完整 |
| `workspace:*` 依赖 | 发布前需转为实际版本号 |
| vte-core description 为空 | npm 页面无描述 |
| 无 CI/CD 发布流程 | 手动发布易出错 |
| 无 CHANGELOG | 用户无法追踪版本变更 |
| VS Code 扩展需用 vsce | 不能用 npm publish |

## 计划

### Step 1: 补全 package.json 字段（7 个包）

为每个 npm 包（vte-core, vte-vite-plugin, vte-cli, vte-compiler, vte-react, vte-playground, vte-language-server）添加：

```json
{
  "files": ["dist"],
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.js"
    }
  },
  "publishConfig": {
    "access": "public"
  }
}
```

同时修复：
- vte-core: 补全 description
- 所有包: 确保 repository/homepage/bugs 字段一致

**涉及文件：**
- `packages/vte-core/package.json`
- `packages/vte-vite-plugin/package.json`
- `packages/vte-cli/package.json`
- `packages/vte-compiler/package.json`
- `packages/vte-react/package.json`
- `packages/vte-playground/package.json`
- `packages/vte-language-server/package.json`

### Step 2: 添加 .npmrc

创建 `.npmrc` 确保 scoped 包默认公开发布：

```
@vte-js:registry=https://registry.npmjs.org/
```

**新建文件：** `.npmrc`

### Step 3: 添加发布脚本到根 package.json

```json
{
  "scripts": {
    "release": "pnpm -r run build && pnpm -r publish --no-git-checks",
    "release:core": "pnpm --filter @vte-js/core run build && pnpm --filter @vte-js/core publish --no-git-checks",
    "release:vite-plugin": "pnpm --filter @vte-js/vite-plugin run build && pnpm --filter @vte-js/vite-plugin publish --no-git-checks",
    "release:cli": "pnpm --filter @vte-js/cli run build && pnpm --filter @vte-js/cli publish --no-git-checks",
    "release:compiler": "pnpm --filter @vte-js/compiler run build && pnpm --filter @vte-js/compiler publish --no-git-checks",
    "release:react": "pnpm --filter @vte-js/react run build && pnpm --filter @vte-js/react publish --no-git-checks",
    "release:playground": "pnpm --filter @vte-js/playground run build && pnpm --filter @vte-js/playground publish --no-git-checks",
    "release:language-server": "pnpm --filter @vte-js/language-server run build && pnpm --filter @vte-js/language-server publish --no-git-checks",
    "release:vscode": "pnpm --filter @vte-js/vscode run build && cd packages/vte-vscode && npx vsce publish",
    "version:patch": "pnpm -r version patch --no-git-tag-version",
    "version:minor": "pnpm -r version minor --no-git-tag-version",
    "version:major": "pnpm -r version major --no-git-tag-version"
  }
}
```

**修改文件：** `package.json`

### Step 4: VS Code 扩展特殊处理

VS Code 扩展使用 vsce 而非 npm publish：
- 不需要 `files`/`exports`/`publishConfig` 字段
- 确保 .vscodeignore 存在（排除源码等）
- 添加 `vsce publish` 脚本

**检查/新建文件：** `packages/vte-vscode/.vscodeignore`

### Step 5: GitHub Actions 自动发布（可选）

创建 `.github/workflows/release.yml`：
- 触发条件：推送 tag（v*）
- 步骤：checkout → install → build → publish
- 使用 NPM_TOKEN secret

**新建文件：** `.github/workflows/release.yml`

## 发布流程

手动发布（推荐先用这个）：
```bash
# 1. 确保所有测试通过
pnpm test

# 2. 构建所有包
pnpm build

# 3. 登录 npm
npm login

# 4. 发布所有包
pnpm release

# 5. 发布 VS Code 扩展（需要 vsce login）
pnpm release:vscode
```

## 验证方式

1. `npm pack --dry-run` 检查每个包的发布内容
2. 确认 dist/ 文件正确包含
3. 确认 source/ 测试文件被排除
4. 在 npmjs.com 确认包已发布
